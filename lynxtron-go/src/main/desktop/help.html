Lynxtron Fiddle

HELP

The easiest way to try Lynxtron — an Electron-style desktop runtime that renders Lynx UI instead of a browser. Write a small fiddle, press Run, and it launches as a real desktop app. No project setup, nothing to configure.

体验 Lynxtron 最简单的方式 —— 一个 Electron 风格的桌面运行时, 用 Lynx 渲染界面而不是浏览器。写一个小小的 fiddle,按下 Run, 它就会作为真正的桌面应用启动。无需搭建工程,无需任何配置。

Quick start快速开始

01 · PICK选模板

Start from Hello Lynxtron (⌘N), or open any showcase from the Gallery as your starting point.

从 Hello Lynxtron 模板开始(⌘N), 或从 Gallery 打开任意 showcase 作为起点。

02 · EDIT写代码

Edit renderer.tsx for the UI and main.js for the window. Panes highlight as you type; toggle files from the sidebar.

在 renderer.tsx 里写界面,在 main.js 里控制窗口。 编辑器实时高亮;侧边栏可以增删和显隐文件。

03 · RUN运行

Press ⌘R. Your fiddle builds and opens in its own window; build output and logs stream into the Console below.

按 ⌘R。fiddle 构建后在独立窗口中打开; 构建输出和运行日志实时流入下方 Console。

A fiddle is a Lynx app, not a web pageFiddle 是 Lynx 应用,不是网页

Lynxtron windows load a compiled Lynx bundle — there is no Chromium, no DOM, and no index.html. Three things to keep in mind:

Lynxtron 窗口加载的是编译后的 Lynx bundle —— 没有 Chromium、没有 DOM、 也没有 index.html。三件事需要记住:

  • UI is ReactLynx: <view> / <text> / <image> elements. All text must live inside <text>. 界面用 ReactLynx 编写:<view> / <text> / <image> 等元素。所有文字必须放在 <text> 里。
  • main.js runs in the main process and calls win.loadFile('main.lynx.bundle') — the bundle the Fiddle builds from your renderer. main.js 运行在主进程,调用 win.loadFile('main.lynx.bundle') —— 也就是 Fiddle 从你的 renderer 构建出的 bundle。
  • CSS is Lynx CSS: flexbox and CSS variables work; browser-isms (:hover layout tricks, overflow: scroll, DOM APIs) don't. CSS 是 Lynx CSS:flexbox 和 CSS 变量可用;浏览器特有的写法 (:hover 布局技巧、overflow: scroll、DOM API)不可用。

What's in the app功能一览

Editor mosaic编辑器网格

Each file is a live native editor pane with syntax highlighting. Add, rename, hide or maximize panes; drag the sashes to resize the grid.

每个文件都是一块带语法高亮的原生编辑器。可以新增、重命名、 隐藏或最大化任意一块;拖动分隔条调整网格布局。

Run & Console

Run writes your edits to a workspace, rebuilds only when the source actually changed, and launches a real process. Stop kills it; the shared Console streams every build and runtime line, with Copy.

Run 会把编辑写入 workspace,仅在源码真正变化时重新构建, 然后启动真实进程;Stop 终止它。共享 Console 实时输出全部构建与运行日志,支持一键复制。

Showcase galleryShowcase 画廊

Built-in examples — from a floating clock to native-texture canvas. Open loads one into the editors, Run launches it as-is, Web serves the cross-platform ones in your browser, IDE opens the workspace in a separate window.

内置示例覆盖悬浮时钟到原生纹理画布。Open 载入编辑器、 Run 直接运行、Web 在浏览器中运行跨端示例、 IDE 在独立窗口打开整个 workspace。

Share via Gist用 Gist 分享

Publish your fiddle as a GitHub Gist and update it in place; load any gist by pasting its URL into the address bar. History lets you check out earlier revisions. (Add a token in Settings → GitHub.)

把 fiddle 发布为 GitHub Gist 并原地更新;把任意 gist 的 URL 粘贴进地址栏即可载入。History 可以检出历史版本。(在 Settings → GitHub 配置 token。)

Runtime versions运行时版本

The version button shows the bundled Lynxtron runtime. Point the chooser at local runtime builds to run your fiddle against a different Lynxtron than the one shipped with the app.

版本按钮显示内置的 Lynxtron 运行时。在版本选择器中指向本地的 运行时构建,即可用不同版本的 Lynxtron 运行你的 fiddle。

Appearance外观

Dark and light themes cover the whole app including the native editors, live from Settings — editor font size too. The welcome tour can be replayed from the Help menu.

深浅两套主题覆盖整个应用(包括原生编辑器),在 Settings 中即时切换, 编辑器字号同样如此。欢迎导览可从 Help 菜单重新播放。

Deep links深链

Other tools (and the docs site) can drive the Fiddle through its URL scheme:

其他工具(以及文档站)可以通过 URL scheme 驱动 Fiddle:

lynxtron://showcase/open?id=counter            open a showcase in the Fiddle
lynxtron://showcase/open?id=counter&target=ide  …in a separate IDE window
lynxtron://lynxview_page?bundle=https://…/x.lynx.bundle   preview a hosted bundle
lynxtron://showcase/open?id=counter            在 Fiddle 中打开 showcase
lynxtron://showcase/open?id=counter&target=ide  ……在独立 IDE 窗口中打开
lynxtron://lynxview_page?bundle=https://…/x.lynx.bundle   预览托管的 bundle
Everything is autosaved: the Fiddle restores your last session — files, layout and edits — every time it starts. Run also writes your edits back into the showcase workspace on disk, so treat opened showcases as working copies. 一切自动保存:每次启动时 Fiddle 都会恢复上次会话 —— 文件、布局和编辑内容。 Run 也会把编辑写回磁盘上的 showcase workspace,请把打开的 showcase 当作工作副本对待。
